# Data Agent Guide
Scope: `/data`.

- Держите `componentAreas` неизменяемыми: уникальные `id` для зон и компонентов, непустые `states`/`tags`, описания в прошедшем PR-чек листе.
- Перед коммитом прогоняйте `npm run test:coverage` — фасадные тесты валидируют карту и поймают пропуски метаданных.
- Для новых логов используйте `src/server/logger.ts`: файл по умолчанию `data/app.log`, формат `ISO [LEVEL] message`.
- Не храните артефакты (`*.log`, временные дампы) под Git; оставляйте только структуру каталогов.

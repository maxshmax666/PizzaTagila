# Быстрый обзор кодовой базы для новичков

- **Старт:** откройте `src/index.ts` — это фасад, который реэкспортирует ядро, данные и токены. Посмотрите тесты в `src/index.test.ts`, чтобы понять ожидаемое поведение.
- **Данные и токены:** компоненты описаны в `src/data/componentAreas.ts`; дизайн-токены — в `src/design/tokens.ts`. Все новые публичные функции должны покрываться тестами.
- **Ядро:** чистые хелперы лежат в `src/core/*` и работают с фасадом. Серверные утилиты — в `src/server` (Node-only).
- **UI:** тонкий слой в `src/App.tsx` и `src/components/*` использует 8pt сетку и палитру `designTokens`.
- **Локальный запуск:** `npm install`, затем `npm run dev`. Проверки перед коммитом: `npm run lint` и `npm run test:coverage`.
- **Логи:** `appendLog` из `src/server/logger.ts` пишет ISO-даты, уровни и сообщения в `data/app.log` (создаётся автоматически).
- **Документация:** читайте `README.md`, `docs/UI_SPEC.md` и `CHANGELOG.md` для истории изменений.
